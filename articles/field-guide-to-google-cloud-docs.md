# はじめに

Google Cloud の知識を得るために公式ドキュメントは欠かせませんが、ドキュメントの読み方がよくわからず上手く活用できていないエンジニアの方々もいるのでは ではないでしょうか？本稿はそんなみなさんにとって Google Cloud 公式ドキュメントを最大限活用し、最大効率で知識を得るための歩き方を説明します。

本稿では、Google Cloud 公式ドキュメントの歩き方として、公式ドキュメントの構造とホームポジション、そしてホームポジションへの辿り着き方を説明します。
また、公式ドキュメントを読むための tips や、他の信頼できるリソースを紹介します。

# ホームポジション

Google Cloud 公式ドキュメントはプロダクトごとに用意されていますが、各プロダクト共通してホームポジションは次の画像のようなページです。

![Cloud Run Home](/images/articles/field-guide-to-google-cloud-docs/home-cloud-run.png)
_Cloud Run ドキュメントのホームポジション_

![BigQuery Home](/images/articles/field-guide-to-google-cloud-docs/home-bigquery.png)
_BigQuery ドキュメントのホームポジション_

ホームポジションとは筆者が勝手に呼んでいるだけで、ここを起点にすれば公式ドキュメントのあらゆる情報にアクセスしやすいページのことです。ドキュメントの構造を理解し、ホームポジションに辿り着くことができれば必要な情報の多くは手に入るでしょう。

各プロダクトのホームポジションは「ガイド」のページです。目印は上部にガイド、リファレンス、サンプル、リソースなどのタブがあり、左部のメニューには公式ドキュメントで扱っている様々なトピックへのリンクが一覧になっています。また、URL は `docs.cloud.google.com/{product}/docs/*` という形式になります。

![Tabs](/images/articles/field-guide-to-google-cloud-docs/tabs.png)

# ドキュメント構造

各プロダクトの公式ドキュメントは概ね共通して次のような構造とコンテンツを持ちます。

- **ガイド**
  - プロダクトの概要説明
  - 必要な概念の説明
  - チュートリアル
  - 設定方法
  - 利用手順
  - ベストプラクティス
  - トラブルシューティング
- **リファレンス**
  - API リファレンス
  - モニタリング指標
  - クライアントライブラリ リファレンス
  - gcloud リファレンス
  - その他、各種設定や DSL 等のリファレンス
- **サンプル**
  - 様々なユースケースにおけるサンプルコードや設定
- **リソース**
  - 料金
  - 割り当てと上限 (quota)
  - リリースノート
  - SLA
  - その他、サポートバージョンなど

多くのケースではガイドの中から知りたい情報を探すことになります。ただし、API リファレンスや料金はガイドではなく別タブにあると覚えておけば迷わずドキュメントを読めるようになります。

:::message
ただ、プロダクトで微妙に構造が違ったり、料金が料金ページになくガイドにしかなかったりするケースもあります。あくまでもだいたいこんな形をしている、ぐらいのものとして覚えておいてください。
:::

# ホームポジションへの辿り着き方

ドキュメント構造を把握してホームポジションへ辿り着くことができれば、欲しい情報にアクセスすることは難しくありません。しかし、ホームポジションに辿り着くまでが難しいというケースがあります。検索しても直接ガイドへのリンクが出なかったり、各プロダクトのホームページからドキュメントへの導線が統一されていないことが、ホームポジションへのアクセスを難しくしている要因です。

以下で説明する2パターンをおさえておけば、だいたいのケースでホームポジションに辿り着くことができるようになります。

## ドキュメントホームから探す

全プロダクトの公式ドキュメントへのリンクを集めたホーム ([docs.cloud.google.com](https://docs.cloud.google.com)) から各プロダクトのホームポジションへアクセスするパターンです。

まずはホームにアクセスして、「本日はどのようなご要件でしょうか？」を無視して「プロダクトを検索」に目的のプロダクト名を入力しましょう。

![Documentation Home](/images/articles/field-guide-to-google-cloud-docs/home.png)

そうするとプロダクト名が表示されるのでクリックします。

![Documentation Home (searched)](/images/articles/field-guide-to-google-cloud-docs/home2.png)

「プロダクトのドキュメントを読む」というボタンがあるのでそれをクリックします。「プロダクトのドキュメントを読む」ボタンがない場合、上部に「ガイド」タブがあるのでそれをクリックしてください。また、この手順を踏まずにホームポジションに辿り着くプロダクトもあります。

![Documentation Home (button)](/images/articles/field-guide-to-google-cloud-docs/home3.png)

## プロダクトホームから探す

プロダクト名で Google 検索をしてドキュメントを探す場合はプロダクトホームが入口になるケースが多いでしょう。

![Google Search](/images/articles/field-guide-to-google-cloud-docs/google-search.png)

プロダクトホームはプロダクトごとに様々でドキュメントの導線も異なりますが、最近は左下にドキュメントへのリンクが用意されているプロダクトが多いのでまずは左下を探しましょう。ドキュメントへのリンクが見つかれば、クリックしてガイドを目指せば大丈夫です。

![Product Home](/images/articles/field-guide-to-google-cloud-docs/product-home.png)

左下にドキュメントへのリンクがないプロダクトホームはおそらくこのような見た目をしているはずです。

![Product Home without link](/images/articles/field-guide-to-google-cloud-docs/product-home2.png)

このようなタイプのプロダクトホームは左メニューの「ドキュメント」をクリックして、下部にある「すべてのプロダクト ドキュメントを見る」というリンクをクリックしてください。

![Product Home link](/images/articles/field-guide-to-google-cloud-docs/product-home3.png)

# 公式ドキュメント以外のリソース

Google Cloud を学んだり調べたりするとき様々なリソースがありますが、まずはそれぞれのリソースの信頼性を理解しておくと良いでしょう。ここで信頼性が高いとは、最新の情報が反映されており正確である、という意味です。また、ここでの信頼性と役に立つかどうかは別軸の基準です。信頼性は低いが役に立つというリソースもあり、そのようなリソースも重要性は高いでしょう。
